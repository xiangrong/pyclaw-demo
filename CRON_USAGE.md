# PyClaw Cron 定时任务使用说明

## 功能概述

PyClaw 的 Cron 定时任务功能允许你创建定时执行的任务。支持以下调度方式：

- **一次性任务**：延迟一段时间后执行（如：30分钟后、2小时后）
- **固定间隔任务**：每隔固定时间执行（如：每30分钟、每2小时）
- **Cron表达式任务**：支持标准Cron表达式（如：每天上午10点）
- **指定时间任务**：在指定的精确时间执行

## 使用方式

### 1. 创建定时任务

使用 `cronjob` 工具创建任务，只需提供 `prompt` 和 `schedule` 参数：

```
帮我创建一个定时任务，每小时提醒我喝杯水
```

```
30分钟后提醒我检查邮件
```

```
每天上午10点提醒我开站会
```

### 2. 查看所有任务

```
列出所有定时任务
```

### 3. 暂停/恢复任务

```
暂停任务 abc12345
```

```
恢复任务 abc12345
```

### 4. 删除任务

```
删除任务 abc12345
```

### 5. 立即触发测试

```
立即执行任务 abc12345
```

## 支持的调度格式

| 格式 | 示例 | 说明 |
|------|------|------|
| 分钟延迟 | `30m` | 30分钟后执行一次 |
| 小时延迟 | `2h` | 2小时后执行一次 |
| 天延迟 | `1d` | 1天后执行一次 |
| 固定间隔 | `every 30m` | 每30分钟执行一次 |
| 固定间隔 | `every 2h` | 每2小时执行一次 |
| Cron表达式 | `0 10 * * *` | 每天上午10点执行 |
| ISO时间 | `2026-05-20T14:00:00` | 指定时间执行 |

## Cron 表达式说明

标准5字段Cron表达式：
```
分钟 小时 日期 月份 星期
0     10    *    *    *   ← 每天上午10点
```

## 安全特性

- ✅ **Prompt注入防护**：自动检测并阻止潜在的prompt注入攻击
- ✅ **不可见字符检测**：检测并阻止包含隐藏Unicode字符的恶意prompt
- ✅ **子进程隔离**：任务在独立子进程中执行，不影响主进程
- ✅ **文件锁**：防止多实例重复执行同一任务

## 执行流程

```
创建任务 → 存入 ~/.pyclaw/cron/jobs.json → 每分钟检查 →
到期执行 → 结果保存到 ~/.pyclaw/cron/output/{job_id}/ →
通过原消息通道发送结果 → 更新任务状态
```

## 命令行工具

## 常见问题

### Q: 任务执行后没有收到回复？
A: 检查任务的 `deliver` 设置是否为 `origin`，确保你的消息平台正常连接。

### Q: 如何让任务执行后不发送消息？
A: 在prompt开头加上 `[SILENT]` 标记，如：`[SILENT] 备份数据库`

### Q: 可以创建多少个任务？
A: 没有硬限制，但建议不超过100个活跃任务以保持性能。

### Q: Agent重启后任务会丢失吗？
A: 不会。任务保存在 `~/.pyclaw/cron/jobs.json` 文件中，重启后自动加载。

### Q: 可以在一个任务中调用另一个定时任务吗？
A: 不可以。为了防止递归攻击，cron任务执行时不允许创建新的定时任务。
